# Session Notes — For Picking Up In a Fresh Claude Session

This document is written specifically for a future AI session picking up this project. It summarises what was built, why, and what's left to do.

---

## What This Repo Is

A community fork of [WoWs-Builder-Team/minimap_renderer](https://github.com/WoWs-Builder-Team/minimap_renderer), which renders World of Warships `.wowsreplay` files into `.mp4` minimap videos. The original was abandoned at WoWs 14.11. This fork revives it for 15.2 and 15.3.

The renderer is a **Python pip package** installed into a venv. Our changes are:
1. Patches applied to the installed packages (via `apply_patches.py`)
2. New version folders added to `replay_unpack/clients/wows/versions/`
3. New data files generated per version in `renderer/versions/`
4. New utility scripts (`update_version.py`, `render_dual.py`, `scripts/`)

## Key Architecture Points

### How Replay Parsing Works

1. `.wowsreplay` → `ReplayReader` (decrypts + decompresses) → raw packet stream
2. Packet stream → `ReplayPlayer` → finds correct version folder → loads `battle_controller.py`
3. `battle_controller.py` subscribes to game events and builds `ReplayData`
4. `ReplayData` → `Renderer` → draws each frame → encodes to MP4

### Version Folder Naming

The game version string in the replay is `"15,3,0,12267945"`. After stripping the last 2 chars and replacing commas: `"15_3_0"`. This is used to find both:
- `replay_unpack/clients/wows/versions/15_3_0/` (parser)
- `renderer/versions/15_3_0/resources/` (data files)

### The GameParams Format Change (15.3)

In 15.3, Wargaming changed `GameParams.data` format:
- Old: standard Python pickle
- New: `%bin` header + remaining bytes reversed + zlib decompressed + pickle with custom classes + wrapped in `{"ASIA": {...}, "NA": {...}, "EU": {...}}`

`scripts/extract_gameparams.py` handles both formats automatically.

### Why abilities.json Is Not Updated

The renderer uses `abilities.json` for per-ship consumable loadout data (which consumables a ship has, their stats). This was generated by the original project's `create_data.py` which we haven't ported to the new GameParams format. Impact: consumable icons for slots 4, 6, 8 (new since 14.11) don't display, and sonar/radar circles for ships not in abilities.json don't render. The render still works — these are gracefully skipped.

## Adding Support For a New Version (Quick Reference)

1. Run `python update_version.py --game "PATH_TO_WOWS" --version X.Y`
2. Test with a recent replay: `python -m render --replay test.wowsreplay`
3. If it crashes with a method signature error, check `PATCHES.md` and update `battle_controller.py`

The most common crash after a version update is a changed method signature in `Avatar.def` or `Vehicle.def`. Compare the new `.def` file with the previous version to find what changed.
